## 当前待办

### 输入框快捷键复制粘贴与占位提示行为修复

- 状态：`已完成`
- 优先级：`P2`
- 创建日期：`2026-05-28`
- 完成日期：`2026-05-28`

#### 背景说明

当前部分输入框里的内容需要支持常规快捷键操作，尤其是 `Command+C` 复制、`Command+V` 粘贴、`Command+A` 全选等系统级文本编辑行为。同时，输入框里的预置提示文字只能作为 placeholder，占位提示不能成为真实内容，也不能影响用户输入、选择、复制、粘贴或提交。

#### 具体实施范围

- 全面检查 App 内所有文本输入控件，包括单行输入、多行输入、搜索框、AI 对话输入、分析目标输入、业务空间背景输入、数据源配置输入、字段/报表描述输入等。
- 确保输入框获得焦点后，系统快捷键 `Command+C / Command+V / Command+X / Command+A / Command+Z / Command+Shift+Z` 正常工作。
- 修复被自定义快捷键、按钮、焦点管理、overlay、placeholder 或容器手势拦截导致的复制粘贴失效问题。
- 预置提示文字必须使用 placeholder / prompt 机制或独立提示层展示，不能写入绑定的真实文本值。
- 当真实内容为空时显示提示文字；用户开始输入后提示文字消失或退到辅助提示位置。
- 复制、粘贴、提交、保存时只处理用户真实输入内容，不包含预置提示文字。
- 多行输入框内容较长时应支持滚动、选择和快捷键编辑，不出现显示不全或无法选中复制的问题。

#### 验收标准

- 在主要输入框中可以用 `Command+A` 全选真实内容、`Command+C` 复制、`Command+V` 粘贴、`Command+X` 剪切。
- 输入框为空时能看到预置提示文字，但复制或提交不会包含提示文字。
- 用户输入后提示文字不遮挡、不混入、不影响光标位置和文本选择。
- 分析目标输入、AI 对话输入、业务空间背景输入至少完成手动验证。
- GUI 回归需要用户授权后执行；默认不自动打包 DMG。

## 后续迭代 Backlog

### 持久化 AI 任务队列完整迭代

- 状态：`已完成`
- 优先级：`P1`
- 创建日期：`2026-05-28`
- 完成日期：`2026-05-28`

#### 背景说明

当前已经有基础版 AI 任务能力：`AIJobQueue` 支持单次请求内自动重试和校验修正，`AIJobRecord` 会记录最近任务状态，`分析资料 > AI 任务` 可以查看任务摘要并对部分任务重试。但这还不是完整的持久化后台任务队列。

目标是把现有能力升级为真正可恢复、可重试、可查看每一步的 AI 任务系统。AI 分析、Memo、报告、机会评分、业务地图、数据源推荐、外部事件影响分析、指标语义抽取等任务失败后可以恢复；App 重启后仍能继续处理未完成任务；配置类错误暂停等待用户处理。

#### 具体实施范围

- 新增 `PersistentAIJob` 持久化任务模型，保存任务类型、目标对象、输入快照、prompt、状态、尝试次数、最大即时重试次数、下次重试时间、错误类型、日志、创建/更新时间。
- 新增全局 `AIJobScheduler`，App 启动后扫描未完成任务并恢复执行。
- 默认同一时间只运行 1 个 AI 任务，避免并发请求导致模型接口压力和 UI 状态混乱。
- 可恢复错误自动重试，包括超时、断网、429、5xx、JSON 格式错误、AI 校验失败。
- 配置类错误暂停等待用户处理，包括 API Key 缺失、BaseURL 错误、模型不存在、余额/权限不足、401、403、404。
- 每个任务先做最多 6 次即时重试；仍失败但属于可恢复错误时，进入后台延迟重试。
- 延迟重试采用指数退避：5 分钟、15 分钟、30 分钟、1 小时、2 小时，之后每 6 小时重试一次，直到成功、用户取消或错误变成不可恢复。
- 下游任务必须等待上游任务完成，例如分析会话成功后才触发机会评分，报告任务必须使用已通过校验的最新分析上下文。
- 改造现有 AI 调用入口，让分析会话、AI Memo、Word 报告结构生成、机会评分、业务地图生成、数据源推荐、外部事件影响分析、指标语义抽取都通过 `AIJobScheduler` 入队。
- 保留 `AIJobQueue` 作为单个任务执行器，继续负责请求、即时重试、校验和自动修正。
- 现有 `AIJobRecord` 作为兼容展示记录保留，主要数据源切换为 `PersistentAIJob`。
- 手动重试使用任务保存的 payload 精确重跑，不再通过最近一条用户消息间接重试。
- `分析会话 > 分析资料 > AI 任务` 升级为真正的任务中心。
- 任务中心展示任务类型、目标、状态、尝试次数、下次重试时间、最近错误。
- 每个任务可展开查看完整日志：请求、校验、失败原因、自动修正、重试等待、完成时间。
- 支持操作：`重试`、`取消`、`复制错误`、`查看输入摘要`。
- 顶部状态提示如果存在暂停任务，明确显示需要处理的问题，例如 API Key、模型不存在、余额不足或 BaseURL 错误。
- 用户点击取消后，任务状态变为 `已取消`，后续不会再自动调度。
- 正在运行的任务需要绑定 Swift `Task` 句柄，取消时尽量中断网络请求；如果请求已经返回，结果不再写入已取消任务。
- App 重启后，`请求中 / 校验中 / 自动修正中` 的任务统一恢复为 `等待中`，再由 Scheduler 重新执行，避免卡死在运行态。

#### 验收标准

- 模拟超时、断网、429、5xx、JSON 格式错误、AI 校验失败时，任务会自动重试并记录完整日志。
- App 重启后，未完成任务能恢复执行。
- API Key 缺失、BaseURL 错误、401/403/404、模型不存在、余额/权限错误时，任务进入 `暂停等待处理`，不无限重试。
- 发送分析需求后，先完成分析会话任务，再触发机会评分任务。
- Memo/报告任务只使用通过校验的最新分析结果。
- `AI 任务` 页面能看到任务状态、尝试次数、下次重试时间、错误和完整日志。
- 手动重试使用原始任务 payload，不依赖最近一条用户消息。
- 取消任务后不会继续写入结果。
- 回归通过：`swift build --disable-sandbox`、`./script/non_gui_regression.sh`、`./script/build_and_run.sh --verify`。
- GUI 回归需用户明确授权，不自动打包 DMG。

### Tableau 底层数据源直连读取

- 状态：`暂缓`
- 优先级：`P1`
- 创建日期：`2026-06-05`

#### 背景说明

未来希望 NexaFlow 可以直接读取 Tableau 中已有的业务数据，减少用户手动从 Tableau 导出表格的操作。目标是通过 Tableau REST API 和 PAT Token 连接 Tableau Cloud / Server，优先读取用户有权限访问的 Published Data Source / Extract 底层数据；如果没有底层下载权限、数据源是 Live Connection 或 Tableau 无法提供底层行数据，则降级为读取 View / Worksheet 的 Crosstab 导出数据。

这项能力只作为未来待办沉淀，当前不新增代码、不接真实 Tableau API、不改变现有导入、分析和报告逻辑。

#### 具体实施范围

- 新增 Tableau 连接配置，支持 Server / Cloud URL、Site、PAT Token Name、PAT Secret。
- PAT Secret 需要存入 Keychain，不写入 workspace，不进入日志。
- Tableau 连接必须绑定业务空间；一个业务空间下的 Tableau 配置、同步记录和导入报表不应出现在其他业务空间。
- 支持测试连接，列出当前用户有权限访问的 Project、Workbook、View 和 Published Data Source。
- 优先下载 Tableau Data Source / Extract；如果下载到 `.tdsx` 或 `.twbx`，需要解包并查找其中的 `.hyper`、`.csv`、`.xlsx` 等数据文件。
- `.hyper` 需要通过独立解析链路转换为 NexaFlow 现有 `CSVTable / ImportedReport` 模型，再进入当前 Data Pack 的报表池。
- 如果底层数据不可读取，需要明确提示原因，例如无 Download 权限、Live Connection 无 extract、PAT 权限不足或对象不存在。
- 底层数据不可读时，提供 View / Worksheet Crosstab 导入作为降级方案，但必须标记为 `视图导出数据`，避免误以为已读取完整底层数据源。
- 导入后的 Tableau 报表继续复用现有表格链路：报表池、选表、AI 读取覆盖、SQL/Notebook 计算证据、老板版报告和 Word 导出。
- AI 回复和报告中必须说明 Tableau 数据来源、Workbook / View / Data Source 名称、同步时间、行列数、权限限制，以及数据是底层 Extract 还是 View 导出。
- Tableau 同步需要写入同步日志，方便回溯每次读取了哪个对象、读取方式、状态、错误和导入结果。

#### 验收标准

- PAT 连接成功后，可以列出当前 Site 下用户有权限访问的 Workbook、View 和 Published Data Source。
- 可下载的 Published Data Source / Extract 能同步为 `ImportedReport`，并可加入当前分析任务。
- `.tdsx / .twbx` 包含 `.hyper` 时，能读取表结构和行数据，并转换成 NexaFlow 报表。
- 无底层下载权限或 Live Connection 无法读取底层行数据时，能给出明确限制，并允许降级导入 View Crosstab。
- 业务空间隔离正确：一个空间的 Tableau 配置、同步记录和报表不会混入其他空间。
- AI 读取范围和老板版报告能展示 Tableau 来源、读取方式、行列数、同步时间和限制。
- 当前只是待办沉淀，不执行 Swift 构建、GUI 回归或 DMG 打包。
